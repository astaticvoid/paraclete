# Paired Session #2 — Theoria legibility build, on glass (USB-C)

Date: 2026-07-10. Driver: Dustin (iPad, USB-C direct link, 3.0 ms RTT).
Build: legibility phase (`7e7a39a`…`19b59b3`), open-mode default, 471 tests.
Context: same-day follow-up to the autonomous legibility session
(`design/phases/theoria-legibility-report.md`); first on-glass judgment.

## Verdict (driver)

**"Improved a lot… it will be the baseline. But no — far below bar."**
The legibility items landed (labels, mode, selection all read), but the
surface concept itself is wrong: an Elektron-style skin layered on a
Launchpad grid mirror. "Chaotic and unfocused — a lot of stuff that shows
with no util in view" (the ~64 dead pads).

## Findings

- **F1 (direction, keystone): the Launchpad-mirror is the wrong foundation
  for Theoria.** The driver's target model: **consistent, fixed inputs with a
  contextual screen window that changes** — the hardware-workstation pattern.
  Not there yet, needs a spike, and needs *more driver input to articulate* —
  design it together, not solo.
- **F2 (direction): park the Launchpad.** The physical-Launchpad path is "a
  good version" — freeze it; stop letting its 9×9 geometry shape Theoria.
  One surface at a time: the Elektron-model tablet surface first.
- **F3 (interaction): no drag-draw.** Programming steps is tap-tap-tap;
  should paint a run of steps in one drag.
- **F4 (interaction): tap-param-then-drag is a bad encoder experience**, and
  the encoder row sits near the top edge so upward drags are cramped. Gesture
  and placement both wrong.
- **F5 (sound, bug): kick ducks on fast re-hit** (direct trigger): strong
  first hit, noticeably quiet if re-hit immediately, strong again after a
  pause. Driver guess "compression?" — no compressor exists; see BUG-023.
- **F6 (sound, bug): kick pitch differs wildly between direct trigger and
  sequencer trigger** — seq hits are "very high pitch, params totally
  disconnected." Root-caused same session: Sequencer default step note is 60
  (C4); engine trigger reference is 36 (C2). See BUG-022.
- **F7 (design process, standing directive): "Don't just make what you think
  to make. Read manuals and design. Interface is everything."** Clone the
  param/context-view *discipline* from a specific Elektron box (to be
  chosen), and take Hydrasynth as the second reference — especially its
  source→effect signal-channel view. Envelopes and LFOs must be first-class,
  visible, editable for everything.

## Roadmap delta

1. **BUG-022 / BUG-023 move first** (user-facing sound correctness is
   baseline, per the July prioritization decision).
2. **Baseline interaction wins** on the current client (survive any pivot):
   drag-draw steps, encoder gesture + placement, hide dead grid area.
3. **W2 is re-scoped: "Theoria native surface", design-first.** Before any
   further W-feature code: a paired reference-design spike — study the chosen
   Elektron box's manual + the Hydrasynth manual, then spec the fixed-input
   rail + contextual window, parameter/env/LFO pages, and the
   source→FX channel view. ADR-032 (view-plugin API) follows that spec.
   Naming policy unchanged: references live in design docs only.
4. **Launchpad parked**: no new Launchpad-first investment; F7-cleanup (s1)
   drops to the trigger backlog.
5. P10 C2+ engine depth unchanged (parallel). P13 remains the keystone where
   per-voice env/LFO routing actually arrives — W2's pages must be designed
   for them ahead of that.

Open question for the driver: **which Elektron box is the reference?**
Proposal: the one in the room (Digitakt II — owned, drum-workflow-shaped);
alternative: a synth-voiced box if the reference should match the
analog/FM engines more than the sampler workflow.
